# Update Patch - 05-05-2026

## Ringkasan

Patch ini memperbaiki dua bagian utama sistem Smart Load Balancer:

1. **Training Gate Q-Learning**
   - Q-Learning tetap aktif untuk routing.
   - Q-table hanya update saat training mode aktif.
   - Sistem tidak belajar dari idle traffic, health check, dashboard polling, atau traffic kecil di luar load test.

2. **xDS Strict Safety Gate**
   - xDS tetap memakai strict mode sebelum mengirim endpoint ke Envoy.
   - Validasi xDS dibuat eksplisit menjadi 4 syarat.
   - Training disabled tidak dianggap sebagai Q-Learning mati.

## 1. Training Gate Q-Learning

Ditambahkan Redis flag:

```bash
qlearning_training_enabled
```

### Enable Training

```bash
redis-cli SET qlearning_training_enabled 1
```

Efek:

```text
- reward dihitung
- Q-table boleh update
- qlearning_history boleh bertambah
- epsilon boleh decay
```

### Disable Training

```bash
redis-cli SET qlearning_training_enabled 0
```

Efek:

```text
- routing tetap berjalan
- current_weights tetap ditulis
- selected_action tetap ditulis
- heartbeat tetap ditulis
- Q-table tidak update
- history training tidak bertambah
- epsilon tidak decay
```

Catatan:

```text
training_enabled = 0 bukan berarti Q-Learning mati.
training_enabled = 0 berarti Q-Learning tetap routing, tetapi tidak belajar.
```

## 2. Perubahan `qlearning/runtime.py`

### Fungsi Baru

Ditambahkan fungsi:

```python
is_training_enabled()
```

Fungsi ini membaca Redis key:

```text
qlearning_training_enabled
```

dan menentukan apakah Q-table boleh diperbarui.

### Perluasan `write_idle_status()`

Fungsi `write_idle_status()` diperluas agar mencatat alasan Q-table tidak di-update.

Status yang dicatat:

```text
TRAINING_DISABLED
IDLE_NO_TRAFFIC
```

Data yang disimpan:

```text
cycle
status
state
metrics
throughput
epsilon
training_enabled
timestamp
reason
```

Tujuannya agar runtime dapat membedakan:

```text
1. Training sengaja dimatikan
2. Training aktif, tetapi traffic belum valid
```

## 3. Perubahan `qlearning/loop.py`

Flow Q-Learning diperbaiki agar training gate dicek sebelum reward dan Q-table update.

Q-table hanya update jika:

```text
training_enabled = 1
throughput > MIN_VALID_THROUGHPUT
```

Jika gate gagal:

```text
- skip reward
- skip Q-table update
- skip qlearning_history
- skip epsilon decay
```

Namun routing tetap berjalan.

---

## 4. Flow Akhir Q-Learning

```text
Observe state
-> Select action
-> Execute routing
-> Write current_weights
-> Wait UPDATE_INTERVAL
-> Observe new state
-> Check training gate
-> Check traffic gate
-> Calculate reward
-> Update Q-table
-> Write monitoring/history
-> Decay epsilon
```

Jika training disabled:

```text
Observe
-> Select action
-> Execute routing
-> Write current_weights
-> Skip reward/Q-update/history/epsilon decay
```

## 5. xDS Strict Safety Gate

File:

```text
xds_server.py
```

xDS tetap memakai strict mode dengan 4 syarat:

```text
Syarat 0: qlearning.service aktif
Syarat 1: qlearning_heartbeat fresh
Syarat 2: current_weights valid
Syarat 3: minimal satu backend weighted reachable
```

Jika semua syarat lolos:

```text
xDS kirim backend endpoint ke Envoy
```

Jika salah satu gagal:

```text
xDS kirim fallback endpoint
```

---

## 6. Perubahan `xds_server.py`

Ditambahkan:

```python
QLEARNING_SERVICE_NAME = "qlearning"
```

Ditambahkan fungsi:

```python
is_qlearning_service_active()
```

Fungsi ini mengecek:

```bash
systemctl is-active --quiet qlearning
```

Tujuannya agar xDS tidak memakai `current_weights` lama jika Q-Learning service sudah mati.

---

## 7. Kontrak Return `get_weights_from_redis()`

Fungsi ini wajib return dua nilai:

```python
return weights_dict, reason_string
```

Contoh valid:

```python
return normalized, "backend_mode_strict"
return FALLBACK_WEIGHTS, "qlearning_service_inactive"
return FALLBACK_WEIGHTS, "no_qlearning_heartbeat"
return FALLBACK_WEIGHTS, "qlearning_heartbeat_timeout"
return FALLBACK_WEIGHTS, "no_current_weights"
return FALLBACK_WEIGHTS, "all_weights_zero"
return FALLBACK_WEIGHTS, "all_weighted_backends_unreachable"
return FALLBACK_WEIGHTS, "unexpected_error"
```

Tidak boleh return satu nilai seperti:

```python
return weights
return normalized
return FALLBACK_WEIGHTS
```

Karena `stream_endpoints()` memakai:

```python
weights, reason = get_weights_from_redis()
```

## 8. Training Gate Tidak Terhubung dengan xDS Fallback

`qlearning_training_enabled` tidak dicek oleh xDS.

Alasannya:

```text
training_enabled hanya mengatur proses belajar Q-table,
bukan menentukan apakah Q-Learning service hidup atau mati.
```

Saat training disabled, xDS tetap mengirim backend selama:

```text
qlearning.service aktif
heartbeat fresh
current_weights valid
backend reachable
```
